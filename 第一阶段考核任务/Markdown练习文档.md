# 我对 Github 的理解

1. 我认为Github是一个大型的代码开源平台。
2. Pull request 是用来向仓库提交自己的代码，有利于多人团队协作，当然 Pull Requset 不等同于直接把仓库的权限给了别人，而是给仓库的主人一个浏览你所提交代码的机会。
3. Fork 的本意就是分支，相当于把别人的仓库复制一遍保留在自己的账户里成为副本，可以在 Fork 的分支里面修改代码，同时给原仓库发 Pull request。
4. 我是在实际使用中一步一步的接触到这些东西的，从最开始的一个小仓库，到和朋友一起做一个大仓库，在实际使用中一步一步学习，从而学会使用这些东西。
